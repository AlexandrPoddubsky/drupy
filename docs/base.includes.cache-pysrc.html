 97    """ 
 98     Store data in the persistent cache. 
 99     
100     The persistent cache is split up into four database 
101     tables. Contributed plugins can add additional tables. 
102     
103     'cache_page': This table stores generated pages for anonymous 
104     users. This is the only table affected by the page cache setting on 
105     the administrator panel. 
106     
107     'cache_menu': Stores the cachable part of the users' menus. 
108     
109     'cache_filter': Stores filtered pieces of content. This table is 
110     periodically cleared of stale entries by cron. 
111     
112     'cache': Generic cache storage table. 
113     
114     The reasons for having several tables are as follows: 
115     
116     - smaller tables allow for faster selects and inserts 
117     - we try to put fast changing cache items and rather static 
118       ones into different tables. The effect is that only the fast 
119       changing tables will need a lot of writes to disk. The more 
120       static tables will also be better cachable with MySQL's query cache 
121     
122     @param cid 
123       The cache ID of the data to store. 
124     @param data 
125       The data to store in the cache. Complex data types will be 
126       automatically serialized before insertion. 
127       Strings will be stored as plain text and not serialized. 
128     @param table 
129       The table table to store the data in. Valid core values are 'cache_filter', 
130       'cache_menu', 'cache_page', or 'cache'. 
131     @param expire 
132       One of the following values: 
133       - CACHE_PERMANENT: Indicates that the item should never be removed unless 
134         explicitly told to using cache_clear_all() with a cache ID. 
135       - CACHE_TEMPORARY: Indicates that the item should be removed at the next 
136         general cache wipe. 
137       - A Unix timestamp: Indicates that the item should be kept at least until 
138         the given time, after which it behaves like CACHE_TEMPORARY. 
139     @param headers 
140       A string containing HTTP php.header information for cached pages. 
141    """ 
142    if expire is None: 
143      expire = lib_bootstrap.CACHE_PERMANENT 
144      fields = { 
145        'serialized' : 0, 
146        'created' : REQUEST_TIME, 
147        'expire' : expire, 
148        'headers' : headers 
149      } 
150    if (not php.is_string(data)):

153    else: 
154      fields['data'] = data 
155      fields['serialized'] = 0 
156    lib_database.merge(table).key({'cid' : cid}).fields(fields).execute()
